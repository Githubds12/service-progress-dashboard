## 1. Executive Summary
Pegasus implements a robust multi-layered security system with advanced anti-automation measures on its mobile application. The platform uses sophisticated device fingerprinting powered by Akamai Bot Manager (`X-acf-sensor-data`), which collects deep device telemetry. The registration flow requires this token for both the SMS initiation and the OTP validation steps. Automation feasibility is extremely low without a mobile device farm or advanced Akamai token generation capabilities.

## 3. Flow Details

### Flow 1: Signup (Registration)

**Step 1: Send SMS OTP**
- **Endpoint**: `POST https://mw.flypgs.com/pegasus/registration`
- **Purpose**: Initialize registration and send SMS OTP
- **Notable Headers**:
    - `X-API-KEY`: Static or slightly rotating API Key
    - `X-PLATFORM`: android
    - `X-VERSION`: 3.69.0
    - `X-DEVICE-ID`: Unique Device UUID
    - `X-acf-sensor-data`: Complex Akamai Bot Manager device fingerprint token
- **Request Payload**:
    ```json
    {
        "name": "Deepanshu",
        "surname": "Singh",
        "phone": {"countryCode": "91", "areaCode": "8791", "number": "267460"},
        "email": "deepanshusinghdigitalheroes@gmail.com",
        "smsSelected": true,
        "emailSelected": true
    }
    ```
- **Response**:
    ```json
    {
        "memberId": "451504966",
        "timer": 3,
        "activationType": "SMS",
        "lastSignupFromDevice": false
    }
    ```
- **Analysis**: Server returns a temporary `memberId` and a `SESSION` cookie. The `timer` field dictates the UI cooldown.

**Step 2: Validate OTP**
- **Endpoint**: `POST https://mw.flypgs.com/pegasus/registration/validate-member-code`
- **Purpose**: Submit the SMS OTP to verify the phone number
- **Notable Headers**:
    - `X-API-KEY`: API Key
    - `X-acf-sensor-data`: Akamai Bot Manager sensor data
- **Request Payload**:
    ```json
    {
        "code": "3333",
        "contactInfo": {
            "phoneNumber": {"countryCode": "39", "areaCode": "3800", "number": "804198"}
        }
    }
    ```
- **Response**:
    ```json
    {
        "title": {"key": "error.title.01016", "values": []},
        "message": {"key": "error.message.01016", "values": []},
        "code": "01016"
    }
    ```
- **Analysis**: Completes the verification. (500 Error observed in capture due to invalid code).

## 4. Security & Reversing Notes

### Encryption/Signing Mechanisms

**1. Device Fingerprinting (Akamai Bot Manager)**
- **Header**: `X-acf-sensor-data`
- **Format**: `6,a,Base64_Payload_Part1,Base64_Payload_Part2$Base64_Signature`
- **Analysis**: Generated by Akamai's SDK. Collects extensive hardware telemetry, OS details, screen properties, and runtime anomalies. Reversing this payload is notoriously difficult.

**2. Session Management**
- **Cookies**: 
  - `SESSION`: Standard backend session tracking cookie.
  - `ak_bmsc`: Akamai Bot Manager session cookie tracking request flows.

### Bot Detection
- **SDK**: Akamai Bot Manager
- **Headers Monitored**: `X-acf-sensor-data`
- **Behavioral Analysis**: Monitors the context of the device and request speed.

### Key Security Features
1. **Akamai Bot Manager**: Primary defense against automated API interactions.
2. **Session binding**: Requests are tied via backend `SESSION` and Akamai cookies.

## 5. Conclusion

### Automation Feasibility: 5%

### Critical Blockers:
1. **Akamai SDK Integration**: Requires real device execution or an advanced headless emulator that perfectly mimics Android telemetry.
2. **X-acf-sensor-data Payload**: Cryptographically secured and constantly evolving format. Standard API scripts will be instantly flagged with 403 Forbidden or silent failures.

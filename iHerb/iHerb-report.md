## 1. Executive Summary
iHerb implements a robust, high-security onboarding flow leveraging multiple enterprise-grade protection layers including **reCAPTCHA Enterprise**, **PerimeterX (PX)**, and **Forter**. The registration process is strictly decoupled into account verification, multi-step OTP validation, and final account creation. The presence of advanced bot detection and fraud prevention systems makes automated interaction significantly challenging without specialized bypass techniques.

## 3. Flow Details

### Flow 1: User Onboarding & Registration
The core flow involves verifying the phone number and completing the profile.

**Step 1: Check Account Existence**
- **Endpoint**: `POST https://checkout2.iherb.com/auth/api/account/exists`
- **Description**: Checks if the mobile number is already registered.
- **Request Body**: `{"username":"+91XXXXXXXXXX","claim":"registerMobile","mobileCountry":"IN"}`

**Step 2: Request OTP**
- **Endpoint**: `POST https://checkout2.iherb.com/auth/api/otp/send`
- **Description**: Triggers the SMS OTP sending process.
- **Mandatory Headers**: `g-recaptcha` (Enterprise token)
- **Request Body**: `{"username":"+91XXXXXXXXXX","claim":"registerMobile","mobileCountry":"IN"}`

**Step 3: Validate OTP**
- **Endpoint**: `POST https://checkout2.iherb.com/auth/api/otp/validate`
- **Description**: Verifies the 6-digit code.
- **Request Body**: `{"username":"+91XXXXXXXXXX","claim":"registerMobile","code":"663349","mobileCountry":"IN"}`
- **Response**: Returns a persistent `state` token for the next step.

**Step 4: Account Creation**
- **Endpoint**: `POST https://checkout2.iherb.com/auth/api/register`
- **Description**: Finalizes the registration with a password and the state token.
- **Request Body**: `{"username":"+91XXXXXXXXXX","password":"...","isSubscribe":true,"mobileCountry":"IN","state":"<state_token>"}`

## 4. Security & Reversing Notes
- **reCAPTCHA Enterprise**: The `g-recaptcha` header in the `otp/send` request is a high-entropy token generated by the Google SDK. Bypassing this requires either high-quality proxy rotation or browser-based automation.
- **PerimeterX (PX)**: Active monitoring via `px-cloud.net`. Any abnormal request patterns or missing device-specific headers (`deviceuuid`, `x-device-id`) will trigger blocking.
- **Forter (Fraud Prevention)**: Used for behavioral analysis. Requests are monitored via `m.api.forter.com`.
- **Identity Server**: iHerb uses a custom identity server (`checkout2.iherb.com/auth`) for OAuth2-compliant authentication.

## 5. Conclusion

### Automation Feasibility: 25%

### Critical Blockers:
1. **reCAPTCHA Enterprise**: Mandatory token for the initial OTP request.
2. **PerimeterX & Forter**: Sophisticated risk-based blocking of non-standard client signatures.
3. **SSL Pinning**: High-level certificate pinning prevents easy MITM analysis on updated app versions.

### Recommendations:
Automation should focus on using high-quality Android emulators or real devices with injected scripts to handle the reCAPTCHA token generation locally before forwarding to the API client.

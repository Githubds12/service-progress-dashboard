## 1. Executive Summary
Klook implements a robust, multi-layered security architecture to protect its authentication and registration flows. The platform utilizes **DataDome** for bot detection and interstitial captcha challenges, supplemented by **Forter** for fraud prevention and behavioral analysis. All critical API requests under the `appapi.klook.com` domain are secured with a proprietary request signing mechanism (`x-signature`) and session-bound identifiers. The authentication flow is strictly stateful, requiring a `trace_id` and a dynamic `otp_token` to be passed between the SMS trigger and verification steps. Automation is highly difficult due to the combination of visual captcha challenges and complex cryptographic request validation.

## 4. Security & Reversing Notes

### Bot Protection (DataDome)
- Klook integrates DataDome to monitor API traffic. Requests that exhibit automated patterns are challenged with an interstitial captcha page (`geo.captcha-delivery.com`).
- The `datadome` cookie is mandatory and must be updated after solving challenges.

### Request Signing (`x-signature`)
- The `x-signature` header is generated by the application for all state-changing requests. It is calculated based on the request URI, body, and the `x-timestamp`.
- Bypassing this requires reverse engineering the native signature logic within the Android APK.

### Fraud Prevention (Forter)
- The app utilizes Forter (`m.api.forter.com`) to collect deep device telemetry and behavioral data (e.g., touch events, accelerometer data) to build a trust profile for the user.

## 5. Conclusion

### Automation Feasibility: 20%

### Critical Blockers:
1. **DataDome Interstitials**: Automated SMS triggering is frequently blocked by DataDome's behavioral firewall, requiring a captcha solver.
2. **Dynamic Request Signing**: The `x-signature` must be correctly recalculated for every request, making static replay attacks impossible.
3. **Behavioral Telemetry**: Forter's background data collection makes it easy to distinguish between a real device and a headless browser or emulator.
